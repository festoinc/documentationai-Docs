## Prerequisites

Before using the BookingPal Supplier API, ensure the following:

<Steps>
  <Step title="Registration" icon="check-circle">
    Your organization is registered and activated with BookingPal.
  </Step>
  <Step title="Kick-off Meeting" icon="users">
    You have completed a kick-off meeting with the BookingPal team.
  </Step>
  <Step title="Questionnaire" icon="file-text">
    The opening questionnaire has been filled out and submitted.
  </Step>
  <Step title="Credentials" icon="key">
    You have your login email and password.
  </Step>
</Steps>

## General Guidelines

<Callout kind="alert" collapsed="false">
All parameters in API requests are **case sensitive** and must be used exactly as documented.
</Callout>

### Payload Limits
- Maximum payload per request: **200 KB**
- Responses are returned as JSON
- Successful `DELETE` requests return `200` with an empty response body

### Authentication
- Every request requires **both** credentials:
  - `x-api-key` **header** — your permanent API key provided by BookingPal
  - `jwt` **query parameter** — session token (valid for **120 minutes**)
- Two token types: **PMS-level** and **PM-level**
- Incoming push/webhook endpoints use `x-api-key` header for verification

### Rate Limits
- Send all changed data for rates/availability in **one request**
- Image operations are processed asynchronously via a **cron job**
- Use unique URLs and filenames for images

### Data Constraints
- Maximum end date for rates/availability/yields: **3 years** from the current month
- LOS pricing: values from **1 to 46 days**
- Security deposits for Airbnb: **100–5,000 USD**
- Fees and taxes: always send the **complete list** (full update)
- Product updates: include all fields you want to **retain**

### Push Notification Retries
- On **504 Timeout** errors, BookingPal retries up to **7 times** with exponential backoff (1s, 2s, 4s, 8s, 16s, 32s, 64s)

### Deprecations
- `POST /push` (New Reservation) → Use General Reservation Notification
- `POST /cancel` (Cancellation) → Use General Reservation Notification
- `DELETE /image` → Use `POST /image/delete`
- `DELETE /image/{productId}` → Use `PUT /image`
