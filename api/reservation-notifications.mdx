## Overview

BookingPal sends reservation notifications to your configured endpoint when bookings are created, updated, or cancelled. You can also query reservation data via GET endpoints.

<Callout kind="info">
- **GET endpoints** require PM-level credentials.
- **Push endpoints** (incoming from BookingPal) do not require auth — verified via `x-api-key` header.
- On a **504 Timeout**, BookingPal retries up to 7 times with exponential backoff (1s, 2s, 4s, 8s, 16s, 32s, 64s).
- Credit card data is included only if the PMS processes payments in their system.
</Callout>

## General Reservation Notification — PUSH

BookingPal sends this POST to your `reservationLink` endpoint for all reservation events.

| Field | Value |
|---|---|
| **Method** | `POST` (incoming to your server) |
| **Auth** | Verified via `x-api-key` header |

### Action Types

| Action | Description |
|---|---|
| `NEW` | New reservation created |
| `UPDATE` | Reservation modified (same reservation ID) |
| `CANCEL` | Reservation cancelled |
| `RESERVATION_REQUEST` | Booking request awaiting approval |
| `RESERVATION_REQUEST_VOIDED` | Booking request cancelled |
