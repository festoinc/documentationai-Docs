## Overview

Product endpoints allow Property Managers to manage their vacation rental properties. All endpoints require **PM-level credentials**.

<Callout kind="alert">
Each request can include only **one product**. When updating, you must include all fields from the original create request that you want to retain — any fields not included will be deleted/overwritten. The `altId` field cannot be updated.
</Callout>

## Create Product

| Field | Value |
|---|---|
| **Method** | `POST` |
| **Endpoint** | `/product` |
| **Auth** | JWT (PM credentials) |

```bash
curl -X POST "https://apidemo.mybookingpal.com/product?jwt=TOKEN" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "name": { "language": "EN", "value": "Beach Villa" },
      "rooms": 3,
      "bathrooms": 2,
      "persons": 6,
      "propertyType": "VILLA",
      "currency": "USD",
      "supportedLosRates": false
    }
  }'
```

## Activate List Product

| Field | Value |
|---|---|
| **Method** | `POST` |
| **Endpoint** | `/product/activation` |
| **Auth** | JWT (PM credentials) |

Properties must be activated before they can be distributed to any channel. Once activated, they are queued for internal BookingPal validation.

<Callout kind="info">
You will receive asynchronous push messages when validation is complete, indicating whether each property is valid or not.
</Callout>

```bash
curl -X POST "https://apidemo.mybookingpal.com/product/activation?jwt=TOKEN" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"data": [1235124634, 1235124636]}'
```

**Response:**

```json
{
  "message": "Property with ids [1235124634, 1235124636] will be put in Queue for validation. Please expect response over push message.",
  "errorMessage": [],
  "is_error": false,
  "code": ""
}
```

## Deactivate List Product

| Field | Value |
|---|---|
| **Method** | `POST` |
| **Endpoint** | `/product/deactivation` |
| **Auth** | JWT (PM credentials) |

Deactivates properties and closes calendars on all channels where they are listed.

```bash
curl -X POST "https://apidemo.mybookingpal.com/product/deactivation?jwt=TOKEN" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"data": [1235124634, 1235124636]}'
```
