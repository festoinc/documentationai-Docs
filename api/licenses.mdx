## Overview

License endpoints allow PMs to manage property license information required by the Booking.com channel. All endpoints require **PM-level credentials**.

## Get License Information

| Field | Value |
|---|---|
| **Method** | `GET` |
| **Endpoint** | `/license/{productId}` |
| **Auth** | JWT (PM credentials) |

<ParamField path="productId" param-type="string" required="true">
  The ID of the property.
</ParamField>

```bash
curl -G "https://apidemo.mybookingpal.com/license/61692799?jwt=TOKEN"
  -H "x-api-key: YOUR_API_KEY" \
```

---

## Create or Update License Information

| Field | Value |
|---|---|
| **Method** | `PUT` |
| **Endpoint** | `/license/{productId}` |
| **Auth** | JWT (PM credentials) |

License details must comply with the requirements obtained from the GET `/license/requirements/{productId}` endpoint.

<ParamField path="productId" param-type="string" required="true">
  The ID of the property.
</ParamField>

```bash
curl -X PUT "https://apidemo.mybookingpal.com/license/61692799?jwt=TOKEN" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": [
      {
        "contentData": [],
        "variantId": "license-variant-id"
      }
    ]
  }'
```
