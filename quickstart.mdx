## Overview

This quickstart will guide you through making your first API call to the BookingPal Supplier API.

<Callout kind="alert">
Before starting, ensure your organization is registered and activated with BookingPal, and you have your login credentials. [Contact us](https://bookingpal.com/en/contact-us/) if you need access.
</Callout>

## Step 1: Authenticate

Obtain a JWT token by calling the Login endpoint with your credentials.

<Steps>
  <Step title="Call the Login endpoint" icon="key">
    Send a `GET` request to `/authc/login` with your email and password as query parameters.
  </Step>
  <Step title="Save the token" icon="save">
    The response includes a `token` field. Use this token in all subsequent requests as the `jwt` query parameter. Tokens are valid for **120 minutes**.
  </Step>
</Steps>

**Example Request:**

```bash
curl -G "https://apidemo.mybookingpal.com/authc/login" \
  --data-urlencode "username=your-email@example.com" \
  --data-urlencode "password=your-password"
```

**Example Response:**

```json
{
  "token": "d5d308244d9eb48a/e9073f6d-85a1-49b8-a799-19c57b66fb24",
  "partyId": 61692799,
  "organizationId": 61690135,
  "name": "Riverwalk Inn",
  "currency": "USD",
  "supplierId": 61692799,
  "is_error": false
}
```

## Step 2: List Your Property Managers

Use the token to fetch your Property Manager list.

```bash
curl -G "https://apidemo.mybookingpal.com/pmlist" \
  -H "x-api-key: YOUR_API_KEY" \
  --data-urlencode "jwt=YOUR_TOKEN_HERE"
```

## Step 3: Create a Property

Create a new product (property) under a Property Manager.

```bash
curl -X POST "https://apidemo.mybookingpal.com/product?jwt=YOUR_TOKEN_HERE" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "name": {"language": "EN", "value": "Beach Villa"},
      "rooms": 3,
      "bathrooms": 2,
      "persons": 6,
      "propertyType": "VILLA",
      "currency": "USD",
      "supportedLosRates": false
    }
  }'
```

## Step 4: Activate the Property

Activate the property so it can be distributed to channels.

```bash
curl -X POST "https://apidemo.mybookingpal.com/product/activation?jwt=YOUR_TOKEN_HERE" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"data": [1235124634]}'
```

## Step 5: Set Up Push Notifications

Configure your webhook endpoints to receive real-time updates.

```bash
curl -X POST "https://apidemo.mybookingpal.com/info?jwt=YOUR_TOKEN_HERE" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "reservationLink": "https://yourserver.com/reservation",
      "asyncPush": "https://yourserver.com/async-push",
      "requestToBook": "https://yourserver.com/request-to-book"
    }
  }'
```

## Step 6: Push Rates and Availability

Send daily rates and availability for your property.

```bash
curl -X POST "https://apidemo.mybookingpal.com/ra?jwt=YOUR_TOKEN_HERE" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "productId": "1235124634",
      "ratesAvailabilities": [
        {
          "date": "2026-06-01",
          "availableCount": 1,
          "minStay": 3,
          "maxStay": 14,
          "rate": 150.00,
          "currency": "USD"
        }
      ]
    }
  }'
```

## Token Types

<Tabs>
  <Tab title="PMS-Level Token" icon="building">
    Used for requests related to PMS or Property Manager data.

    **Endpoints:** Property Manager CRUD, PM List, Push Notifications, Channel Configuration
  </Tab>
  <Tab title="PM-Level Token" icon="user">
    Used with PM credentials for managing properties.

    **Endpoints:** Products, Rates & Availability, Images, Fees & Taxes, Reservations, Messaging
  </Tab>
</Tabs>
