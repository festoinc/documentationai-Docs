## API Updates

<Update label="2026-04-28" description="General Reservation Notification" tags={["feature"]}>
## What changed
- Added **General Reservation Notification** endpoint (`POST /reservation`) that consolidates new, update, and cancel reservation events.
- Deprecated separate `POST /push` and `POST /cancel` endpoints.
- Added `reservationLink` field to Push Notification configuration.
- Added `x-api-key` header verification for incoming reservation pushes.
- Added retry mechanism: up to 7 retries with exponential backoff on 504 errors.
</Update>

<Update label="2026-04-28" description="Asynchronous Push Messages" tags={["feature"]}>
## What changed
- Added async push notifications for **fees and taxes** imports (`POST /pushFeeTax`).
- Added async push notifications for **LOS rates** imports (`POST /pushlos`).
- Added async push notifications for **rates & availability** imports (`POST /pushra`).
</Update>

<Update label="2026-04-28" description="Image Management" tags={["feature", "breaking"]}>
## What changed
- Added `POST /image/delete` for deleting specific images (replaces deprecated `DELETE /image`).
- Added `PUT /image` for full image update (replaces deprecated `DELETE /image/{productId}`).
- Deprecated `DELETE /image` and `DELETE /image/{productId}` endpoints.
</Update>

<Update label="2026-04-28" description="Validation Settings" tags={["feature"]}>
## What changed
- Added Fee and Tax Mandatory at Property Level endpoints.
- `GET /validation-settings/get-validation` ‚Äî retrieve mandatory settings.
- `POST /validation-settings/import-update` ‚Äî set mandatory flags.
- `POST /validation-settings/removed` ‚Äî reset to defaults.
</Update>

<Update label="2026-04-28" description="Request to Book" tags={["feature"]}>
## What changed
- Added `POST /requestToBookTest` for testing the request-to-book flow.
- Added decline reason codes for request-to-book answers.
</Update>

<Update label="2026-04-28" description="Messaging" tags={["feature"]}>
## What changed
- Added `POST /message/allThreadsSearch` for advanced message search.
- Added test message endpoints (`/messageTest/*`) for development.
</Update>

<Update label="2026-04-28" description="Licenses" tags={["feature"]}>
## What changed
- Added Booking.com license management endpoints.
- `GET /license/{productId}` ‚Äî retrieve license information.
- `PUT /license/{productId}` ‚Äî create/update license information.
- `GET /license/requirements/{productId}` ‚Äî retrieve license requirements.
</Update>
